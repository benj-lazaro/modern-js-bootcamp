null
----
- It represents the intentional absence of any object value
- It MUST be assigned to a variable; the explicit value of nothingness
- It is one of JavaScript's primitive values
- It is treated as falsy for boolean operations
- Reference (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/null)


undefined
---------
- It represents the idea of the lack of value
- It is one of JavaScript's primitive values
- Normally, it is the value that is returned by JavaScript

- A variable that is NOT assigned a value is of type undefined
- A function returns undefined if a value was NOT returned
- Reference (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/undefined)


Example
-------
> let loggedInUser = null;  // Explicitly assigns nothing to the variable loggedInUser
